# Ask Sage: AI-Enabled Analyst Support Program

**U.S. Army Reserve 84 Training Command | Built on the Ask Sage Platform**

---

## Overview

This repository contains the strategic, operational, and training documentation for the AI-Enabled Analyst Support Program — a capability built from scratch to integrate generative AI into the U.S. Army Reserve's training evaluation process.

The program was designed, built, and deployed by a single AI Integration Lead over a 9-month period (March–December 2023), with subsequent field validation during Operation Sentinel Justice (June 2026). It leverages the [Ask Sage](https://asksage.ai/) platform — a FedRAMP-authorized, DoD-compliant generative AI capability — to enhance the efficiency, speed, and quality of assessment products generated by Observer/Controllers (OC/Ts) and Training Assessment Framework (TAF) Analysts.

## Problem

The Army Reserve's training evaluation process had not fundamentally changed in decades. Senior evaluators performed observation synthesis, doctrinal analysis, narrative assessment, and product finalization entirely by hand. During peak exercise tempo, analyst throughput became the primary bottleneck — degrading the timeliness and quality of feedback reaching training audiences.

There was no enterprise AI capability, no infrastructure, no policy framework, and no institutional precedent for integrating generative AI into this mission set.

## Program Architecture

The program is built on four interdependent components:

| Component | Purpose |
|-----------|---------|
| **Standard Operating Procedure (SOP)** | Battle rhythm integration defining evaluator interaction with the platform across all exercise phases |
| **Prompt Library** | Mission-specific, pre-built prompts calibrated to TAF analytical tasks — doctrinal analysis, observation synthesis, AAR development, trend identification |
| **Persona Framework** | Distinct AI behavioral profiles configured per evaluator role (OC/T, TAF Analyst, Senior Mentor, AAR Facilitator) |
| **Data-Cloud Structure** | Organizational backbone connecting users, data, and outputs — structured to mirror actual exercise operations |

## Key Design Principles

- **Integration over replacement** — AI augments existing workflows rather than demanding new processes
- **Methodology over tooling** — program value lives in the methodology, not the platform; all components are transferable
- **Training-first adoption** — no capability deploys without a corresponding training program
- **Measure everything** — every deployment includes structured data collection for validation
- **Document for replication** — every artifact is sufficient for adoption without the original builder present
